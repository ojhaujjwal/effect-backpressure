# Plan: Shared `ojhaujjwal/oxlint-rules` Package

## Architecture

Following the `@effect/oxc` pattern from `repos/effect-smol/packages/tools/oxc/`:

- One standalone npm package that exports an oxlint plugin + optional shared base config
- Both `tesla-charger` and `media-diy-backend` consume it as `"jsPlugins": ["ojhaujjwal/oxlint-rules/oxlint"]`

## Step-by-Step Implementation

### 1. Initialize Repo

Create new empty repo. `pnpm init`. Set `"type": "module"`.

### 2. `package.json` ‚Äî Exports

```jsonc
{
  "name": "ojhaujjwal/oxlint-rules",
  "version": "0.1.0",
  "type": "module",
  "exports": {
    "./package.json": "./package.json",
    "./oxlint": "./src/oxlint/index.ts",           // dev: raw TS
    "./oxlint/rules/*": "./src/oxlint/rules/*.ts", // dev: individual rules
    "./oxlintrc.json": "./oxlintrc.json"           // shared base config
  },
  "publishConfig": {
    "exports": {
      "./package.json": "./package.json",
      "./oxlint": "./dist/oxlint/index.js",        // published: compiled JS
      "./oxlint/rules/*": "./dist/oxlint/rules/*.js",
      "./oxlintrc.json": "./oxlintrc.json"
    }
  },
  "files": ["oxlintrc.json", "src/**/*.ts", "dist/**/*.js", "dist/**/*.d.ts"],
  "scripts": {
    "build": "tsc -b tsconfig.json",
    "test": "vitest"
  },
  "dependencies": {
    "@oxlint/plugins": "^1.68.0"
  },
  "peerDependencies": {
    "typescript": ">=5.0.0"
  },
  "devDependencies": {
    "@types/node": "^22",
    "oxlint": "^1.68.0",
    "typescript": "^5.8",
    "vitest": "^3"
  }
}
```

Key points:
- `@oxlint/plugins` for `defineRule` / `definePlugin` (just type helpers ‚Äî optional but nice)
- `peerDependencies.typescript` since rules are .ts source exported
- `oxlint` as devDependency for testing rules (the `oxlint` types used in test utils)
- Or: use a raw default export (no `@oxlint/plugins` dep) ‚Äî both work

### 4. Plugin Entry Point (`src/oxlint/index.ts`)

```ts
import { definePlugin } from "@oxlint/plugins"  // or raw `export default { ... }`
import rule1 from "./rules/no-effect-ignore.js"
// ... import all 23 rules ...

export default definePlugin({
  meta: { name: "effect-lint" },   // pick a namespace
  rules: {
    "no-effect-ignore": rule1,
    // ... register all rules ...
  }
})
```

### 5. Port All Rules

Copy rules from both projects into `src/oxlint/rules/`. The **11 shared rules** are drop-in ‚Äî choose one canonical copy. For `no-process`, use the media-diy message (references Effect Config). For tesla-charger-only rules, port directly:

| Source | Rule | File |
|--------|------|------|
| Both (11) | `no-effect-ignore`, `no-effect-catchallcause`, `no-effect-asvoid`, `no-silent-error-swallow`, `no-service-option`, `no-nested-layer-provide`, `pipe-max-arguments`, `no-disable-validation`, `no-void-expression`, `no-oxlint-disable`, `no-process` | Copy once |
| media-diy only (4) | `no-effect-any`, `no-async-await`, `no-satisfies`, `no-raw-date` | From media-diy |
| tesla-charger only (8) | `no-console`, `no-node-imports`, `no-unknown-effect-error`, `no-plain-it-with-effect`, `no-vitest-modifiers`, `no-ts-ignore`, `no-ts-nocheck`, `prefer-option-from-nullable` | From tesla-charger |

Total: **23 rules** in the shared package.

For `no-raw-date` ‚Äî it's registered in media-diy's plugin but **not enabled** in `.oxlintrc.json`. Keep it in the shared package; consuming projects choose whether to enable.

### 6. Test Infrastructure

Adapt `@effect/oxc`'s `test/utils.ts`:

```ts
// test/utils.ts
import type { CreateRule, Visitor } from "oxlint"

export const createTestContext = (sourceCode: string) => {
  const errors: Array<{ message: string }> = []
  return {
    errors,
    context: {
      id: "test/rule",
      filename: "/test/file.ts",
      physicalFilename: "/test/file.ts",
      cwd: "/test",
      options: [],
      sourceCode: { text: sourceCode, getText: (n: any) => sourceCode.slice(n?.range?.[0] ?? 0, n?.range?.[1]) },
      report: (opts: { message: string }) => { errors.push(opts) }
    }
  }
}
```

Write tests per rule. Example for `no-process`:

```ts
// test/no-process.test.ts
import { describe, test, expect } from "vitest"
import rule from "../src/oxlint/rules/no-process.js"
import { createTestContext } from "./utils.js"
import { parse } from "oxlint/parser"  // or use oxc-parser

// ... parse source, call rule.create(context).MemberExpression(node), check errors
```

### 7. Optional: Shared Base Config (`oxlintrc.json`)

Include an `oxlintrc.json` with common enabled rules and sensible defaults. Consuming projects `"extends": ["ojhaujjwal/oxlint-rules/oxlintrc.json"]` to get defaults, then override per-project.

### 8. Migrate Consuming Projects

In both `media-diy-backend` and `tesla-charger`:

1. **Remove** `lint/` directory (the custom rules and `index.ts`)
2. **Remove** `@oxlint/plugins` from devDependencies (no longer needed directly)
3. **Add** `"ojhaujjwal/oxlint-rules": "^0.1.0"` to devDependencies
4. **Update** `.oxlintrc.json`:
   - Change `"jsPlugins": ["./lint/index.ts"]` ‚Üí `"jsPlugins": ["ojhaujjwal/oxlint-rules/oxlint"]`
   - Change rule prefixes from `media-diy/` / `tesla-charger/` ‚Üí `effect-lint/`
   - Keep project-specific overrides (files whitelisted, etc.)
5. Run `pnpm lint` and verify zero violations
