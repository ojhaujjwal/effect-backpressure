# Disallow `process.argv` in `no-process` rule

## Summary
Remove the `argv` exemption from the `no-process` rule so that `process.argv` is flagged as an error, just like `process.env.FOO`, `process.cwd()`, etc.

## Changes

### 1. `src/oxlint/rules/no-process.ts` (source)
- Remove the `argv` guard at lines 29-31:
  ```ts
  if (propertyName === "argv") {
    return;
  }
  ```

### 2. `test/no-process.test.ts` (test)
- Move `process.argv` from `valid` to `invalid` with an appropriate error matcher:
  ```ts
  {
    code: "process.argv",
    errors: [{ message: /process/ }]
  }
  ```

## Verification
- Run `npx vitest run test/no-process.test.ts` to confirm the rule catches `process.argv` and still passes all other cases.
